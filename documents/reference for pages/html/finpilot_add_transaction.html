Design decisions

Bottom sheet pattern

The drag handle at the top signals this slides up from the home screen — no full navigation needed. One tap on + opens it, swipe down dismisses.

Red / Green toggle

The active type (Expense or Income) colors the entire amount and the save button. The user never has to think about which mode they're in.

Big centered amount

The number is the primary action — it should dominate. Users type it first, then fill the rest. The blinking cursor invites immediate input.

Category grid

Icon grid beats a dropdown — faster to tap, no scrolling, visually consistent with category colors used throughout the app.

Violet = active field

Focused input borders and icons stay violet — same rule as the auth screen. One color for "where you are right now."

Income has fewer categories

Salary, Freelance, Investment, Other — income sources are simpler than expense categories, so the grid is smaller and less cognitive load.
