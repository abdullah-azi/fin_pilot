How it works

5 frequency types all covered

Once, Hourly, Daily, Monthly, Yearly — the pill row adapts the amount label and preview card to match whichever is selected.

Hourly needs extra inputs

Hrs/day and days/week appear only when Hourly is selected. The app multiplies these to estimate a monthly figure for the AI baseline.

Preview card builds trust

Always shows the per-month equivalent — that's what FinPilot's AI uses internally. User sees the math before saving so there's no surprise.

Once ≠ recurring

One-time income (bonus, sale, gift) is logged but does not update the monthly baseline the AI uses for spending and savings analysis.

Amount label changes

"RATE (PKR/HR)", "AMOUNT (PKR/DAY)", "AMOUNT (PKR/MONTH)" — the label above the big number updates live so the user always knows what they're entering.

Backend: store raw + frequency

Never store only the computed monthly value. Store the raw amount, frequency, and hrs/day + days/week for hourly — so it can be recalculated if the user edits later.
